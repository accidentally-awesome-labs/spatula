<objective>
Ship the SSE infrastructure (AUTH-01) and extend the single-use stream token to SSE (AUTH-02). Two halves: (1) make `RedisEventPublisher.publish` dual-publish — keep the existing pub/sub `PUBLISH` (the WS path is untouched) and add `XADD` to a Redis Stream `jobs:{id}:events`; (2) build the `apps/api/src/sse/` handler that consumes the `?token=` stream token, replays buffered events via `XRANGE`, then tails live events via `XREAD BLOCK` on a dedicated Redis connection, emitting `replay_truncated` when the client's `Last-Event-ID` is too old and a keep-alive comment every 15s.

Purpose: This is the core deliverable of Phase 17. A browser `EventSource` can subscribe to live job events, survive a disconnect, and resume from where it left off.
Output: A dual-publish change in `packages/queue`, a new `apps/api/src/sse/` module (types + buffer + handler), the SSE route mounted in `app.ts` outside the scope-guard chain, an OpenAPI doc tweak on `ws-token`, and the timeout-middleware override that keeps SSE connections alive.

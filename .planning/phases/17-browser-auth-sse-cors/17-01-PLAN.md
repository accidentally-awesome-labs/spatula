    1. Edit `packages/db/src/schema/api-keys.ts`. Add two columns inside the `pgTable('api_keys', {...})` object after `revokedAt`:
       - `supersedes: uuid('supersedes')` — nullable, self-referential FK. Because Drizzle cannot reference a table inside its own definition cleanly, declare it as a plain `uuid('supersedes')` column WITHOUT a `.references()` call in TS; the FK constraint is added in the SQL migration directly.
       - `supersededExpiresAt: timestamp('superseded_expires_at', { withTimezone: true })` — nullable. This records the grace-window expiry copied onto the NEW key's row so the rotate response (D-16) can return `supersededExpiresAt` without a second query.
    2. Create `packages/db/drizzle/0001_api_key_rotation.sql` with statement-breakpoint markers matching `0000_v1_baseline.sql` style:
       ```sql
       ALTER TABLE "api_keys" ADD COLUMN "supersedes" uuid;
       --> statement-breakpoint
       ALTER TABLE "api_keys" ADD COLUMN "superseded_expires_at" timestamp with time zone;
       --> statement-breakpoint
       ALTER TABLE "api_keys" ADD CONSTRAINT "api_keys_supersedes_api_keys_id_fk" FOREIGN KEY ("supersedes") REFERENCES "public"."api_keys"("id") ON DELETE no action ON UPDATE no action;
       ```
    3. Append a new entry to `packages/db/drizzle/meta/_journal.json` `entries` array: `{ "idx": 1, "version": "7", "when": <current epoch ms>, "tag": "0001_api_key_rotation", "breakpoints": true }`. Do NOT modify entry 0.
    4. Generate the matching `packages/db/drizzle/meta/0001_snapshot.json` by copying `meta/0000_snapshot.json` and adding the two columns + the FK to the `api_keys` table entry. If `drizzle-kit generate` is available and produces a clean diff, prefer that — but verify the produced SQL matches step 2 (drizzle-kit sometimes omits self-FKs; if so, hand-edit per step 2).

    The migration runs under `__drizzle_migrations_oss` (Phase 15 two-journal model — do NOT touch the saas journal). This is an additive expand-only change, conforming to the expand-contract-only policy in `docs/runbooks/upgrade.md`.

    Per CONTEXT.md D-05 (planner's discretion on normalizer location — choosing standalone shell script for reuse + reviewability):

    1. Create `scripts/normalize-schema-dump.sh` (executable). It accepts a pg_dump output on stdin and produces a normalized output on stdout. Normalization steps (per CONTEXT.md "Specifics"):
       ```bash
       #!/usr/bin/env bash
       # scripts/normalize-schema-dump.sh
       # Normalizes pg_dump --schema-only output for byte-equivalence diffing.
       # Strips:
       #   - migration-journal rows (any __drizzle_migrations* table COPY/INSERT)
       #   - pg_dump version header comments
       #   - timestamp lines / "Dumped from database version" / "Dumped by pg_dump"
       #   - Empty comment-only lines
       # Sorts: top-level CREATE statements lexically by table name.

       set -euo pipefail

       # Drop pg_dump preamble + timestamp lines + comment-only lines + migration journal data
       sed -E \
         -e '/^-- Dumped (from|by) /d' \
         -e '/^-- PostgreSQL database dump /d' \
         -e '/^-- Started on /d' \
         -e '/^-- Completed on /d' \
         -e '/^SET (statement_timeout|lock_timeout|idle_in_transaction_session_timeout|client_encoding|standard_conforming_strings|xmloption|client_min_messages|row_security)/d' \
         -e '/^SELECT pg_catalog\.set_config/d' \
         | awk '
           # Skip COPY blocks targeting __drizzle_migrations* (data, not schema; not part of equivalence).
           /^COPY (public\.)?__drizzle_migrations/ { skipping = 1; next }
           skipping && /^\\\.$/ { skipping = 0; next }
           skipping { next }
           { print }
         '
       ```
       Make executable: `chmod +x scripts/normalize-schema-dump.sh`

       NOTE: pg_dump --schema-only typically doesn't emit data rows, so the COPY-skipping block is belt-and-suspenders. If executor finds it produces noisy false-positives, simplify.

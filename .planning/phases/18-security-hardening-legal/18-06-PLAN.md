<objective>
Build the DSR (Data Subject Rights) deletion + import server surface: a new BullMQ
`tenant-delete` worker that cascades a GDPR-complete deletion across every tenant-scoped
table + content-store + forensic blobs, the async `DELETE /api/v1/admin/tenants/:id`
route (202 + status reference), and the `POST .../import` route symmetric with export.

Purpose: A public data-tooling product must let an operator fully delete a tenant's data
on request. The cascade must be idempotent + fail-loud (D-09) — GDPR-complete deletion is
the bar, no log-and-continue. The audit log keeps a tombstone (D-08) proving the deletion.
Output: `tenant-delete-worker.ts`, `TenantDataRepository`, `DELETE` + `import` admin routes.
</objective>

<objective>
Ship forensic provenance — `forensic:true` content-store tagging + `suspicious_extraction`
DLQ logging on suspicious/off-schema extractions — and the `GET /api/v1/admin/forensic/extractions`
endpoint (the sole v1 experimental surface) reachable only via `client.experimental.forensic.*`.

Purpose: When the prompt-injection scanner (Plan 18-01) flags a suspicious extraction, the
raw HTML must be archived for forensic review. The forensic endpoint gives admins read
access to that archive with signed URLs and no inline HTML (preventing log-injection).
Output: `forensic-archiver.ts`, `admin-forensic.ts` route, `admin:forensic:read` scope,
OpenAPI experimental tag, `client.experimental.forensic.*` SDK surface.
</objective>

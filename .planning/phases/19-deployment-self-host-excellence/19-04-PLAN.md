<objective>
Create the `deploy/k8s/` kustomize tree (DEPLOY-01): a base (namespace, secrets-with-placeholders, migrate Job, api+worker Deployments, api Service) plus dev and prod overlays. The dev overlay (D-09) ships throwaway in-cluster Postgres + Redis pods so `kubectl apply -k deploy/k8s/overlays/dev` comes up healthy and self-contained on a fresh `kind` cluster.      
      ```yaml
      apiVersion: v1
      kind: Secret

      stringData:

        # add other required SPATULA_* / secrets per .env.example
      ```
      Add a comment block documenting external-secrets / sealed-secrets as upgrade paths (D-08) — NOT requirements.
    - `migrate-job.yaml`: `kind: Job` name `spatula-migrate`, `backoffLimit: 3`, `restartPolicy: OnFailure`, container image `ghcr.io/accidentally-awesome-labs/spatula/migrate:latest` (overridden by overlays), `envFrom: [{ secretRef: { name: spatula-secrets } }]`.     - `api-deployment.yaml`: `kind: Deployment` name `spatula-api`, 1 replica (base), container image `ghcr.io/accidentally-awesome-labs/spatula/api:latest`, `envFrom: [{ secretRef: { name: spatula-secrets } }]`, port 3000, plus probes:
      ```yaml
      startupProbe:    { httpGet: { path: /health/ready, port: 3000 }, failureThreshold: 30, periodSeconds: 10, initialDelaySeconds: 20 }
      livenessProbe:   { httpGet: { path: /health,       port: 3000 }, periodSeconds: 15 }
      readinessProbe:  { httpGet: { path: /health/ready, port: 3000 }, periodSeconds: 10 }
      ```
      The startupProbe gives the migrate Job a head start (Pitfall #6) — no RBAC/initContainer-job-watch needed. Set `terminationGracePeriodSeconds: 30` (matches SHUTDOWN_TIMEOUT_MS 25s + buffer).
    - `worker-deployment.yaml`: `kind: Deployment` name `spatula-worker`, image `.../worker:latest`, `envFrom` secretRef, no HTTP ports/probes (worker serves no HTTP); set `terminationGracePeriodSeconds: 30`.
    - `api-service.yaml`: `kind: Service` name `spatula-api`, selector matching api Deployment, port 3000 -> targetPort 3000, type ClusterIP.
    - `kustomization.yaml`: `apiVersion: kustomize.config.k8s.io/v1beta1, kind: Kustomization, namespace: spatula, resources: [namespace.yaml, secrets.yaml, migrate-job.yaml, api-deployment.yaml, api-service.yaml, worker-deployment.yaml]`.

    All Deployments must set `securityContext: { runAsNonRoot: true, runAsUser: 65532 }` to match the distroless `nonroot` uid from Plan 02.

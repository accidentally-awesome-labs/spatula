# Phase 19.1 Plan 02: ALS Usage Recording (Gap-3 Fix) Summary

**One-liner:** AsyncLocalStorage-backed LLMUsageRecorder wired onto the raw OpenRouter client, recording every worker LLM call to `llm_usage` with race-safe per-job attribution via ALS context wrapping in the three LLM-using handlers.

## What Was Built

### `packages/queue/src/usage-context.ts`

New module providing:

- `AsyncLocalStorage<UsageContext>` exported as `usageContext`
- `UsageContext` interface: `{ tenantId: string; jobId: string }`
- `currentUsageContext(): UsageContext | undefined` helper — reads the active ALS store

The ALS store is the carrier that lets `AlsUsageRecorder.record()` know which tenant/job is active without any parameter threading through the orchestrator call chain.

### `packages/queue/src/als-usage-recorder.ts`

`AlsUsageRecorder implements LLMUsageRecorder`:

- Constructor takes `LlmUsageRepository` (injected)
- `record(usage)`: reads `currentUsageContext()` — if no active context, logs a warning and returns (no throw, no DB write)
- If context present: calls `repo.insert({ tenantId, jobId, model, promptTokens, completionTokens, totalTokens, costUsd: usage.costUsd.toFixed(6), purpose })` fire-and-forget with `.catch(warn)`
- Mirrors the `DefaultUsageRecorder` pattern from `apps/api/src/services/usage-recorder.ts` exactly, except tenant/job come from ALS instead of the constructor

### `packages/queue/src/als-usage-recorder.test.ts`

4 unit tests (all green):

1. **Per-context attribution** — `record()` inside `usageContext.run({T1,J1})` inserts with `tenantId:'T1'`, `jobId:'J1'`
2. **Concurrent scope isolation** — two interleaved `Promise.all` scopes each call `record()` after a microtask yield; each insert receives the correct tenant/job (no cross-attribution)
3. **Graceful no-context skip** — `record()` outside any `usageContext.run` does not throw and does not call `insert`
4. **6-dp costUsd string** — `costUsd: 0.1234567` → `insert` receives `'0.123457'`

### `packages/queue/src/worker-entrypoint.ts`

Three changes:

**1. New imports:**

```typescript
import { LlmUsageRepository } from '@spatula/db'; // added to existing @spatula/db import line
import { usageContext } from './usage-context.js';
import { AlsUsageRecorder } from './als-usage-recorder.js';
```

**2. setUsageRecorder wiring** (after the `built`/`deps` block):

```typescript
// Wire LLM usage recording onto the RAW client (CircuitBreaker wrapper does
// not expose setUsageRecorder). ALS gives race-safe per-job attribution.
if (built?.rawClient && 'setUsageRecorder' in built.rawClient) {
  const llmUsageRepo = new LlmUsageRepository(db);
  (built.rawClient as { setUsageRecorder: (r: AlsUsageRecorder) => void }).setUsageRecorder(
    new AlsUsageRecorder(llmUsageRepo),
  );
  logger.info('LLM usage recorder wired (ALS per-job attribution)');
}
```

The `built?.rawClient` guard means this is a no-op when `_opts.deps` was passed (test-injection path), keeping the test suite clean.

**3. Three handler wraps:**

| Handler          | Before                                                          | After                                                                                                                                           |
| ---------------- | --------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| Crawl            | `await processCrawlJob(job.data, deps)`                         | `await usageContext.run({ tenantId: job.data.tenantId, jobId: job.data.jobId }, () => processCrawlJob(job.data, deps))`                         |
| Schema evolution | `await processSchemaEvolutionJob(job.data, deps, redisForLock)` | `await usageContext.run({ tenantId: job.data.tenantId, jobId: job.data.jobId }, () => processSchemaEvolutionJob(job.data, deps, redisForLock))` |
| Reconciliation   | `await processReconciliationJob(job.data, deps)`                | `await usageContext.run({ tenantId: job.data.tenantId, jobId: job.data.jobId }, () => processReconciliationJob(job.data, deps))`                |

Export/webhook/cleanup/tenant-delete handlers are NOT wrapped (they make no LLM calls).

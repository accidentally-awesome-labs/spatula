<objective>
Close Gap 2 (per-job LLM config). `StaticExtractor`/classifier/schemaEvolver/reconciler/linkEvaluator resolve the model via `resolveModel(this.config, ...)` where `this.config` is the CONSTRUCTION-time `llmConfig` (proven: static-extractor.ts:76,105). The worker builds these once in `buildWorkerDeps` (Plan 01) with a default model → each job's `llm.primaryModel` (the sizing tiers) is ignored. This plan derives the 5 config-dependent components PER JOB inside each LLM-using handler from `job.config.llm`, reusing the shared `llmClient`, crawler, repos, contentStore, and queues.

Purpose: Without this, all three sizing tiers (fast/primary/smart) would record under the same default model — EXEC-06's tier-distinct cost/page is impossible and per-job model pinning is broken.
Output: `deriveJobDeps()` helper, per-job derivation in the crawl/schema/reconciliation handlers, the shared `llmClient` threaded from Plan 01's `built` into the handlers, and a unit test proving the job's model wins.
</objective>

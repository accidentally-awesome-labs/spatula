# Phase 19.1 Plan 01: Worker DI (Gap-1 Fix) Summary

**One-liner:** buildWorkerDeps() composition helper wires full WorkerDeps into startWorker(), fixing the root cause of every queued job DLQ-ing with "WorkerDeps not initialized".

## What Was Built

### `packages/queue/src/build-worker-deps.ts`

New async composition helper that:

1. **Fail-loud first**: throws `buildWorkerDeps: OPENROUTER_API_KEY is required` before any async operations if the env var is unset/empty.
2. Constructs all 11 repositories from the injected `db` (JobRepository, CrawlTaskRepository, PageRepository, ExtractionRepository, SchemaRepository, EntityRepository, EntitySourceRepository, SourceTrustRepository, ExportRepository, ActionRepository, TenantRepository).
3. Selects content store: `CONTENT_STORE=s3` → `createContentStore({...})` with S3 env vars; default → `new PgContentStore(db)` (mirrors `apps/api/src/main.ts` lines 85–96 exactly).
4. Creates base `llmConfig: LLMConfig = { primaryModel: process.env.SPATULA_DEFAULT_MODEL ?? 'deepseek/deepseek-v4-pro' }`.
5. Creates `rawClient` via `createLLMClient({ provider: 'openrouter', ... })` and wraps as `llmClient = new CircuitBreakerLLMClient(rawClient)`.
6. Constructs all 5 LLM-dependent components: PageClassifier, StaticExtractor (base jobId=''), SchemaEvolverImpl, DataReconcilerImpl, LLMLinkEvaluator.
7. Constructs crawl infra: RobotsTxtChecker, InMemoryDomainRateLimiter, CrawlCompletionChecker.
8. Creates crawler via `CrawlerFactory.create({ type: process.env.SPATULA_CRAWLER ?? 'playwright', firecrawlApiKey: process.env.FIRECRAWL_API_KEY })` with a code comment documenting the distroless→Firecrawl requirement.
9. Logs `{ crawlerType, contentStore: contentStoreType, defaultModel }` at startup.

**Return shape** (the seams downstream plans consume):

```typescript
{
  deps: WorkerDeps;
  rawClient: LLMClient;
  llmClient: LLMClient;
  llmConfig: LLMConfig;
}
```

### `packages/queue/src/worker-entrypoint.ts`

Three changes:

1. Added import: `import { buildWorkerDeps } from './build-worker-deps.js';` and `import type { BuildWorkerDepsResult } from './build-worker-deps.js';`

2. Replaced `let deps: WorkerDeps | undefined;` with:

```typescript
let built: BuildWorkerDepsResult | undefined;
let deps: WorkerDeps;
if (_opts?.deps) {
  deps = _opts.deps;
} else {
  built = await buildWorkerDeps({ db, pool, queues });
  deps = built.deps;
}
```

3. `built` kept in local scope so Plans 02/03 can reference `built.rawClient` (for `setUsageRecorder`) and `built.llmClient` / `built.llmConfig` (for per-job overrides) without reopening this file unnecessarily.

The `if (!deps) throw 'WorkerDeps not initialized'` guards in each handler remain as harmless defensive guards — they are now dead code in production but keep TS happy.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Test mock strategy: no importOriginal for @spatula/core**

- **Found during:** Task 1 verification (build-worker-deps.test.ts isolated) was green, but Task 2 full suite run showed tests 1-2 timing out (5s) and tests 3-4 failing with 0 calls on CrawlerFactory.create.
- **Issue:** Using `vi.mock('@spatula/core', async (importOriginal) => { ... })` in a test file that runs in the same vitest worker pool as `tests/unit/exports.test.ts` (which loads the real `@spatula/core`) caused module cache collisions. The `importOriginal` triggered a real `@spatula/core` load including Playwright's `chromium.launch()`, causing the 5s timeout.
- **Fix:** Replaced `importOriginal` with a fully synthetic mock factory. The synthetic mock covers exactly the exports `build-worker-deps.ts` imports — no real module load required.
- **Files modified:** `packages/queue/src/build-worker-deps.test.ts`
- **Commit:** 70aba07

**2. [Rule 1 - Bug] worker-entrypoint.test.ts needed mock for build-worker-deps.js**

- **Found during:** Task 2 — after wiring buildWorkerDeps into startWorker(), the existing worker-entrypoint test (Test 3) threw `buildWorkerDeps: OPENROUTER_API_KEY is required` because the test calls `startWorker()` without `_opts.deps` and without `OPENROUTER_API_KEY` in env.
- **Fix:** Added `vi.mock('../../src/build-worker-deps.js', ...)` to the existing test file returning a stub `BuildWorkerDepsResult`.
- **Files modified:** `packages/queue/tests/unit/worker-entrypoint.test.ts`
- **Commit:** 70aba07
